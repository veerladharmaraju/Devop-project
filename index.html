🚀 Day 6: Connecting Git Bash to EC2 Instance

```

📌 Objective

Successfully connected local system to AWS EC2 Ubuntu instance using SSH via Git Bash and performed basic server setup.

🛠️ Steps Performed

1. EC2 Instance Setup

  • Launched Ubuntu EC2 instance
  • Downloaded .pem key file
  • Enabled SSH (Port 22)

2. Opened Git Bash

Used Git Bash terminal to connect with EC2 instance.

3. Navigated to Key File Location

cd ~/Downloads

4. Changed File Permission

chmod 400 day6-key.pem

5. Connected to EC2 via SSH

ssh -i day6-key.pem ubuntu@YOUR-PUBLIC-IP

6. First-Time Authentication

Accepted connection prompt by typing:

yes

7. Successfully Logged into EC2

Accessed Ubuntu terminal remotely.

8. Installed Packages

sudo apt update sudo apt install nginx -y

9. Checked Nginx Status

systemctl status nginx

10. Verified Web Server

Accessed EC2 public IP in browser to confirm Nginx running.

📚 Key Learnings

  • SSH connection using PEM key
  • Linux file permissions
  • Remote server access
  • Basic web server setup (Nginx)

🎯 Outcome

Successfully established secure connection between local machine and EC2 instance and deployed a basic web server.

🔗 Next Step

Proceed to Day 7: GitHub + EC2 Deployment for live project hosting.
```
